# Big O

## Definition

Big O describes the upper bound of an algorithm's growth rate.

It focuses on scalability instead of actual execution time.

---

## Examples

O(1)

Hash lookup

O(log n)

Binary Search

O(n)

Linear Search

O(n log n)

Merge Sort

O(n²)

Bubble Sort

---

## Advantages

- Easy comparison
- Predicts scalability

---

## Disadvantages

Does not include

- CPU cache
- Memory allocation
- Branch prediction
- Constant factors
